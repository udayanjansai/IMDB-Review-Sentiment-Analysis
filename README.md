# Movie Review Sentiment Analysis with Deep Learning (SimpleRNN, LSTM, GRU)

This project implements and compares three different Recurrent Neural Network (RNN) architectures – SimpleRNN, Long Short-Term Memory (LSTM), and Gated Recurrent Unit (GRU) – for movie review sentiment analysis. The goal is to classify movie reviews as either positive or negative. The best-performing model (GRU) is then integrated into an interactive Streamlit web application for real-time sentiment prediction.

## Project Overview

Movie reviews are a rich source of sentiment data. Understanding the sentiment expressed in reviews can be valuable for various applications, from recommendation systems to market research. This project delves into deep learning techniques to automatically classify the sentiment of IMDb movie reviews.

## Features

*   **Data Loading & Preprocessing:** Utilizes the IMDb movie review dataset, including tokenization, special token handling, and sequence padding.
*   **Model Implementation:** Builds and trains three distinct RNN models: SimpleRNN, LSTM, and GRU.
*   **Model Comparison:** Evaluates and compares the performance (accuracy, loss, parameters) of all three models.
*   **Real-time Testing:** Provides a utility to test custom movie reviews against the trained models.
*   **Streamlit Deployment:** Deploys the best-performing model (GRU) as an interactive web application, allowing users to input reviews and get instant sentiment predictions with confidence scores.

## Models Implemented

1.  **SimpleRNN:** A basic recurrent neural network, serving as a baseline.
2.  **LSTM (Long Short-Term Memory):** An advanced RNN capable of learning long-term dependencies, mitigating the vanishing gradient problem.
3.  **GRU (Gated Recurrent Unit):** A simplified version of LSTM, offering comparable performance with fewer parameters and faster training.

## Dataset

The project uses the **IMDb Movie Review Dataset**, which consists of 50,000 movie reviews, equally split into 25,000 for training and 25,000 for testing. Each set contains 12,500 positive and 12,500 negative reviews, ensuring a balanced dataset.

*   **Reviews:** Sequences of words (represented as integers).
*   **Labels:** Binary (0 for negative, 1 for positive).

## Setup and Installation

To run this project, you'll need Python 3 and the following libraries. It's recommended to use a virtual environment.

1.  **Clone the repository (or download the notebook/files):**
    ```bash
    git clone <your-repository-url>
    cd movie-sentiment-analysis
    ```

2.  **Install dependencies:**
    The `requirements.txt` file lists all necessary packages.
    ```bash
    pip install -r requirements.txt
    ```
    (The `requirements.txt` includes `tensorflow`, `streamlit`, `numpy`, `matplotlib`, `seaborn`, `pyngrok`)

3.  **Download Models (if not already present):**
    The trained Keras models (`simple_rnn_model.keras`, `lstm_model.keras`, `gru_model.keras`) are required for the Streamlit app. If you've run the Colab notebook, they will be saved to your local environment. Otherwise, you'd typically find them in a `models/` directory or need to re-train.

## Running the Streamlit Application

To run the interactive sentiment analysis web application:

1.  **Ensure `app.py` is in your working directory.** (This file is generated by the Colab notebook.)

2.  **Set up ngrok (Optional, for public access):**
    If you want to access your Streamlit app from outside your local machine (e.g., share a link), you'll need `ngrok`. Create an account on [ngrok.com](https://ngrok.com) and get your authtoken.
    If running in Google Colab, you can store your `NGROK_AUTH_TOKEN` in Colab's secrets.

3.  **Run Streamlit:**
    ```bash
    streamlit run app.py
    ```
    This will open the application in your web browser, typically at `http://localhost:8501`. If you configured `ngrok`, it will also provide a public URL.

## Project Structure

*   `movie_sentiment_analysis.ipynb` (or `.py` equivalent): The main notebook/script containing data loading, preprocessing, model building, training, evaluation, and Streamlit file generation.
*   `app.py`: The Streamlit application script for interactive sentiment analysis.
*   `requirements.txt`: Lists all Python dependencies.
*   `*.keras`: Saved Keras model files (e.g., `simple_rnn_model.keras`, `lstm_model.keras`, `gru_model.keras`).

## Results and Analysis

Through experimentation, the **GRU model** consistently achieved the highest test accuracy (approximately 87.15%) on the IMDb dataset, demonstrating superior performance over LSTM (85.71%) and SimpleRNN (52.42%). GRUs also offer a good balance of performance and computational efficiency, making them suitable for larger datasets.
